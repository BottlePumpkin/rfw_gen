# rfw_gen v0.4 Design Spec — Material 위젯 + 이벤트 핸들러

## Overview

rfw_gen v0.2에서 구축한 38개 core 위젯 위에 **이벤트 핸들러 아키텍처**와 **Material 위젯 18개**를 추가한다. GestureDetector(v0.2에서 defer)도 포함.

## Scope

### 신규 위젯 (18개)

**Core (1)**: GestureDetector

**Material (17)**: Scaffold, AppBar, ListTile, Card, Material, ElevatedButton, TextButton, OutlinedButton, FloatingActionButton, InkWell, Divider, VerticalDivider, CircularProgressIndicator, LinearProgressIndicator, Drawer, OverflowBar, Slider

### 신규 아키텍처

- **이벤트 핸들러**: `set state.x = value` + `event "name" { ... }` 두 패턴
- **RfwHandler 헬퍼 클래스**: 빌드타임 타입 안전 핸들러 표현
- **namedSlots ChildType**: Scaffold, AppBar, ListTile용 다중 named child slot
- **자동 import 수집**: core.widgets / material import 자동 결정

## 1. RfwHandler 헬퍼 클래스

### 파일: `packages/rfw_gen/lib/src/rfw_handler.dart`

```dart
/// RFW 이벤트 핸들러를 Dart에서 표현하기 위한 헬퍼.
///
/// RFW는 두 가지 핸들러 패턴을 지원:
/// 1. `set state.field = value` — 위젯 내부 상태 변경
/// 2. `event "name" { ... }` — Flutter 호스트 앱으로 이벤트 전달
class RfwHandler {
  RfwHandler._();

  /// 상태 변경 핸들러 (정적 값): `set state.field = value`
  static RfwSetState setState(String field, dynamic value) =>
      RfwSetState(field, value);

  /// 상태 변경 핸들러 (콜백 인자 값): `set state.field = args.value`
  /// Slider.onChanged 등 콜백에서 전달되는 값을 상태에 저장할 때 사용.
  static RfwSetStateFromArg setStateFromArg(String field,
      [String argName = 'value']) =>
      RfwSetStateFromArg(field, argName);

  /// 이벤트 전달 핸들러: `event "name" { key: value, ... }`
  static RfwEvent event(String name, [Map<String, dynamic> args = const {}]) =>
      RfwEvent(name, args);
}

/// `set state.field = value` 핸들러 표현.
class RfwSetState {
  final String field;
  final dynamic value;
  const RfwSetState(this.field, this.value);
}

/// `set state.field = args.argName` 핸들러 표현.
/// 콜백 인자를 상태에 매핑 (예: Slider.onChanged).
class RfwSetStateFromArg {
  final String field;
  final String argName;
  const RfwSetStateFromArg(this.field, [this.argName = 'value']);
}

/// `event "name" { ... }` 핸들러 표현.
class RfwEvent {
  final String name;
  final Map<String, dynamic> args;
  const RfwEvent(this.name, [this.args = const {}]);
}
```

### 사용 예시

```dart
@RfwWidget('button')
Widget buildButton() {
  return GestureDetector(
    onTap: RfwHandler.setState('pressed', true),
    onLongPress: RfwHandler.event('cart.add', {'itemId': 42}),
    child: Text('Buy'),
  );
}
```


## 3. ExpressionConverter 확장

### 핸들러 변환 메서드

기존 `convert()` 메서드와 별도로 `convertHandler()` 메서드를 추가. AST Visitor가 핸들러 파라미터임을 알 때만 호출.

```dart
/// 핸들러 표현식을 IR 핸들러 값으로 변환.
/// RfwHandler.setState(...), RfwHandler.event(...),
/// RfwSetState(...), RfwEvent(...) 를 인식.
IrValue convertHandler(Expression expr) {
  if (expr is! MethodInvocation) {
    throw UnsupportedExpressionError(
      'Handler must be RfwHandler.setState() or RfwHandler.event()',
      offset: expr.offset,
    );
  }

  final target = expr.target;
  final methodName = expr.methodName.name;

  // RfwHandler.setState/setStateFromArg/event
  if (target is SimpleIdentifier && target.name == 'RfwHandler') {
    if (methodName == 'setState') return _convertSetState(expr);
    if (methodName == 'setStateFromArg') return _convertSetStateFromArg(expr);
    if (methodName == 'event') return _convertEvent(expr);
  }

  // Direct constructors
  if (target == null && methodName == 'RfwSetState') {
    return _convertSetState(expr);
  }
  if (target == null && methodName == 'RfwSetStateFromArg') {
    return _convertSetStateFromArg(expr);
  }
  if (target == null && methodName == 'RfwEvent') {
    return _convertEvent(expr);
  }

  throw UnsupportedExpressionError(
    'Unknown handler expression: $methodName',
    offset: expr.offset,
  );
}

IrSetStateValue _convertSetState(MethodInvocation expr) {
  final args = expr.argumentList.arguments;
  // First arg: field name (string)
  final field = (args[0] as SimpleStringLiteral).value;
  // Second arg: value (any supported expression)
  final value = convert(args[1]);
  return IrSetStateValue(field, value);
}

IrSetStateFromArgValue _convertSetStateFromArg(MethodInvocation expr) {
  final args = expr.argumentList.arguments;
  final field = (args[0] as SimpleStringLiteral).value;
  final argName = args.length > 1
      ? (args[1] as SimpleStringLiteral).value
      : 'value';
  return IrSetStateFromArgValue(field, argName);
}

IrEventValue _convertEvent(MethodInvocation expr) {
  final args = expr.argumentList.arguments;
  // First arg: event name (string)
  final name = (args[0] as SimpleStringLiteral).value;
  // Optional second arg: map of args
  if (args.length > 1) {
    final mapExpr = args[1];
    if (mapExpr is SetOrMapLiteral) {
      final entries = <String, IrValue>{};
      for (final entry in mapExpr.elements) {
        if (entry is MapLiteralEntry) {
          final key = (entry.key as SimpleStringLiteral).value;
          entries[key] = convert(entry.value);
        }
      }
      return IrEventValue(name, entries);
    }
  }
  return IrEventValue(name);
}
```

## 4. RfwtxtEmitter 확장

`_emitValue`에 새 IR 타입 핸들링 추가:

```dart
IrSetStateValue v => 'set state.${v.field} = ${_emitValue(v.value, indent: indent)}',
IrSetStateFromArgValue v => 'set state.${v.field} = args.${v.argName}',
IrEventValue v => _emitEvent(v, indent: indent),
```

```dart
String _emitEvent(IrEventValue event, {required int indent}) {
  if (event.args.isEmpty) {
    return 'event "${event.name}"';
  }
  final buffer = StringBuffer();
  buffer.write('event "${event.name}" ');
  // Inline map for event args
  buffer.write('{ ');
  final entries = event.args.entries.toList();
  for (var i = 0; i < entries.length; i++) {
    buffer.write('${entries[i].key}: ${_emitValue(entries[i].value, indent: indent)}');
    if (i < entries.length - 1) buffer.write(', ');
  }
  buffer.write(' }');
  return buffer.toString();
}
```

## 6. AST Visitor 확장

`_processNamedArgument`에서 핸들러와 namedSlots 처리:

```dart
void _processNamedArgument(
  NamedExpression arg,
  WidgetMapping mapping,
  Map<String, IrValue> properties,
) {
  final paramName = arg.name.label.name;
  final expression = arg.expression;

  // 1. Handler params
  if (mapping.handlerParams.contains(paramName)) {
    try {
      properties[paramName] = expressionConverter.convertHandler(expression);
    } on UnsupportedExpressionError {
      // Silently skip
    }
    return;
  }

  // 2. Named child slots
  if (mapping.childType == ChildType.namedSlots &&
      mapping.namedChildSlots.containsKey(paramName)) {
    final isList = mapping.namedChildSlots[paramName]!;
    if (isList && expression is ListLiteral) {
      final children = expression.elements
          .map((e) => _convertWidget(e as Expression))
          .toList();
      properties[paramName] = IrListValue(children);
    } else if (!isList) {
      properties[paramName] = _convertWidget(expression);
    }
    return;
  }

  // 3. Regular child/children (existing logic)
  if (mapping.childParam != null && paramName == mapping.childParam) {
    // ... existing child handling ...
  }

  // 4. Regular params (existing logic)
  // ...
}
```

## 7. Converter — 자동 import 수집

### 파일: `packages/rfw_gen/lib/src/converter.dart`

`convertFromAst()`에서 하드코딩된 `{'core.widgets'}` 대신 IR 트리에서 import를 수집:

```dart
String convertFromAst(FunctionDeclaration function) {
  // ... existing visitor + irTree extraction ...

  final imports = _collectImports(irTree);
  return emitter.emit(widgetName: widgetName, root: irTree, imports: imports);
}

Set<String> _collectImports(IrWidgetNode node) {
  final imports = <String>{};
  final mapping = registry.supportedWidgets[node.name];
  if (mapping != null) imports.add(mapping.import);

  for (final value in node.properties.values) {
    if (value is IrWidgetNode) {
      imports.addAll(_collectImports(value));
    } else if (value is IrListValue) {
      for (final item in value.values) {
        if (item is IrWidgetNode) imports.addAll(_collectImports(item));
      }
    }
  }
  return imports;
}
```

## 13. 구현 순서

1. IR 확장 + RfwtxtEmitter 확장 (IrValue는 sealed이므로 반드시 동시 수정) + 테스트
2. RfwHandler/RfwSetState/RfwSetStateFromArg/RfwEvent 헬퍼 클래스
3. ExpressionConverter — convertHandler() + 테스트
4. WidgetMapping 확장 (handlerParams, namedSlots, ChildType.namedSlots)
5. AST Visitor — handler + namedSlots 처리 + 테스트
6. Converter — import 자동 수집 + 테스트
7. GestureDetector 위젯 등록 + 테스트
8. Material 위젯 등록 (17개) + 테스트
9. 예제 앱 (3개)
10. Integration 테스트

**Note**: Step 1에서 IR + Emitter를 동시에 해야 하는 이유: `IrValue`는 `sealed class`이므로 새 서브타입 추가 시 `RfwtxtEmitter._emitValue()`의 exhaustive switch가 컴파일 에러를 발생시킨다.
